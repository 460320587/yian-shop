**分层职责说明**：

| 层级 | 对应目录 | 职责 | 与 Java 版的差异 |
|------|---------|------|-----------------|
| 表现层 | `Http/Controllers` | 接收 Request → 调用 Action → 返回 Resource | 无 `@Controller` 注解，路由在 `routes/` 中注册 |
| 验证层 | `Http/Requests` | FormRequest 自动验证，失败自动 422 | 替代 Java `@Valid` + `BindingResult` |
| 响应层 | `Http/Resources` | API Resource 转换模型为 JSON | 替代 Java DTO/VO，但更简单灵活 |
| 业务层 | `Domains/*/Actions` | 单一职责业务动作类 | 替代 Java `ServiceImpl`，无接口默认分离 |
| 数据层 | `Domains/*/Models` | Eloquent 模型 + 关联 + Scope + Observer | 替代 Java Entity + Mapper + DAO 三层 |
| 事件层 | `Events` / `Listeners` | 领域事件解耦模块间通信 | 替代 Java Spring EventBus |
| 队列层 | `Jobs` | 异步任务 | 替代 Java `@Async` + ThreadPool |

## 4.2 领域模块划分

怡安印刷商城共划分为 12 个业务领域，每个领域在 `app/Domains/` 下独立目录，包含完整的 Model + Action + Policy + Event：

| 领域目录 | 负责模块 | 核心模型 |
|---------|---------|---------|
| `Domains/User` | 客户、子账号、地址、企业认证 | `Customer`, `SubAccount`, `CustomerAddress`, `CustomerBrand` |
| `Domains/Product` | 品类、商品、参数、价格 | `ProductCategory`, `Product`, `ProductParam`, `ProductPrice` |
| `Domains/Order` | 订单、明细、状态机、评价 | `Order`, `OrderItem`, `OrderDelivery`, `OrderComment` |
| `Domains/Payment` | 支付流水、退款、钱包、发票 | `Payment`, `Refund`, `CustomerWallet`, `Invoice` |
| `Domains/Logistics` | 配送、快递、云仓、门店 | `ExpressTrack`, `CloudWarehouse`, `Store` |
| `Domains/Vip` | VIP、成长值、积分、优惠券 | `VipLevel`, `GrowthLog`, `PointLog`, `Coupon` |
| `Domains/Notification` | 站内信、短信、邮件、微信模板 | `MessageTemplate`, `Notification`, `CustomerBadge` |
| `Domains/Auth` | 登录、Token、权限、角色 | `AdminUser`, `Role`, `Permission` |
| `Domains/AfterSale` | 售后单、退货、补印、投诉工单 | `AfterSale`, `AfterSaleRating`, `Ticket`, `TicketReply` |
| `Domains/Platform` | 电商代发、平台店铺、平台订单 | `PlatformShop`, `PlatformOrder`, `PlatformSyncLog` |
| `Domains/Admin` | 后台管理、审计日志、数据追踪 | `AuditLog`, `DataChangeLog`, `SystemConfig` |
| `Domains/Common` | 共享内核：枚举、异常、Trait | — |

**模块间通信规则**：

- **同步查询**：一个领域的 Action 可直接注入其他领域的 Eloquent Model 进行只读查询（如订单领域查询客户信息）。
- **状态变更**：一个领域修改另一个领域的数据时，必须通过**领域事件**异步通知，禁止直接修改其他领域的模型。

```php
<?php

// Order 领域创建订单后，通过事件通知 Payment 领域创建待支付流水
namespace App\Domains\Order\Events;

use App\Domains\Order\Models\Order;
use Illuminate\Foundation\Events\Dispatchable;
use Illuminate\Queue\SerializesModels;

class OrderCreated
{
    use Dispatchable, SerializesModels;

    public function __construct(
        public readonly Order $order
    ) {}
}

// Payment 领域监听事件
namespace App\Domains\Payment\Listeners;

use App\Domains\Order\Events\OrderCreated;
use App\Domains\Payment\Actions\CreatePendingPaymentAction;

class CreatePendingPayment
{
    public function __construct(
        private CreatePendingPaymentAction $action
    ) {}

    public function handle(OrderCreated $event): void
    {
        $this->action->execute($event->order);
    }
}
```

```php
// EventServiceProvider.php 中注册监听
protected $listen = [
    \App\Domains\Order\Events\OrderCreated::class => [
        \App\Domains\Payment\Listeners\CreatePendingPayment::class,
        \App\Domains\Notification\Listeners\SendOrderCreatedNotification::class,
        \App\Domains\Logistics\Listeners\ReserveDeliveryCapacity::class,
    ],
];
```

```php
<?php

namespace App\Domains\Common\Enums;

enum OrderSign: int
{
    case IS_NEW = 1 << 0;          // 新单
    case IS_REORDER = 1 << 1;      // 补单
    case IS_LARGE = 1 << 2;        // 大订单
    case NO_AFTER_SALE = 1 << 3;   // 不售后
    case ONE_KEY_REORDER = 1 << 4; // 一键翻单
    // ... 共 25 个标记位

    /**
     * 检查位掩码是否包含当前标记
     */
    public function in(int $flags): bool
    {
        return ($flags & $this->value) !== 0;
    }

    /**
     * 添加标记到位掩码
     */
    public function addTo(int $flags): int
    {
        return $flags | $this->value;
    }

    /**
     * 从位掩码中移除标记
     */
    public function removeFrom(int $flags): int
    {
        return $flags & ~$this->value;
    }
}
```

## 4.4 Laravel Service Container 与依赖注入

Laravel 的 Service Container 是整个框架的核心，负责类的实例化与依赖解析。与 Java Spring 的 `@Autowired` + `@Component` 不同，Laravel 采用**构造函数注入 + 自动解析**机制，无需任何注解。

### 4.4.1 自动解析（Automatic Resolution）

当类构造函数声明了类型提示的参数时，Laravel 容器会自动尝试解析并注入实例：
容器解析流程：

1. 接收到 `CreateOrderAction` 的解析请求。
2. 通过 PHP Reflection 读取构造函数参数列表。
3. 对每个参数递归解析（如 `CreateOrderAction` 依赖 `OrderRepository`，容器继续解析 `OrderRepository`）。
4. 所有依赖解析完成后，实例化 `CreateOrderAction` 并返回。

### 4.4.2 接口绑定（Interface Binding）

当需要切换实现（如支付网关支持微信支付/支付宝）时，在 ServiceProvider 中绑定接口到具体实现：

```php
<?php

namespace App\Providers;

use App\Domains\Payment\Contracts\PaymentGateway;
use App\Domains\Payment\Gateways\AlipayGateway;
use App\Domains\Payment\Gateways\WechatPayGateway;
use Illuminate\Support\ServiceProvider;

class PaymentServiceProvider extends ServiceProvider
{
    public function register(): void
    {
        $this->app->bind(PaymentGateway::class, function ($app) {
            return match (config('payment.default')) {
                'wechat' => new WechatPayGateway(
                    config('payment.wechat.app_id'),
                    config('payment.wechat.mch_id'),
                    config('payment.wechat.key'),
                ),
                'alipay' => new AlipayGateway(
                    config('payment.alipay.app_id'),
                    config('payment.alipay.public_key'),
                    config('payment.alipay.private_key'),
                ),
                default => throw new \InvalidArgumentException('Unsupported payment gateway'),
            };
        });
    }
}
```

使用接口注入：

```php
<?php

namespace App\Domains\Payment\Actions;

use App\Domains\Payment\Contracts\PaymentGateway;

class ProcessPaymentAction
{
    public function __construct(
        private readonly PaymentGateway $gateway, // 自动注入配置中绑定的实现
    ) {}

    public function execute(array $data): PaymentResult
    {
        return $this->gateway->createCharge($data);
    }
}
```

### 4.4.3 单例绑定（Singleton）

对于无状态服务或配置类，使用单例避免重复实例化：

```php
<?php

namespace App\Providers;

use App\Services\Pricing\PricingEngine;
use Illuminate\Support\ServiceProvider;

class AppServiceProvider extends ServiceProvider
{
    public function register(): void
    {
        // 计价引擎：无状态，全局单例
        $this->app->singleton(PricingEngine::class, function ($app) {
            return new PricingEngine(
                $app->make(PriceRuleRepository::class),
                $app->make(DiscountCalculator::class),
            );
        });

        // 上下文绑定：同一接口在不同场景下注入不同实现
        $this->app->when(OrderController::class)
            ->needs(PricingEngine::class)
            ->give(fn () => new PricingEngine(
                $app->make(PriceRuleRepository::class),
                $app->make(EnterpriseDiscountCalculator::class), // 企业客户专用折扣
            ));
    }
}
```

### 4.4.4 方法注入（Method Injection）

不仅构造函数可以注入，控制器方法、闭包、Job 的 `handle` 方法均可注入：

```php
<?php

namespace App\Http\Controllers\Api;

use App\Domains\Product\Models\Product;
use App\Services\Pricing\PricingEngine;
use Illuminate\Http\Request;

class ProductController extends Controller
{
    /**
     * 方法注入：容器自动解析 PricingEngine
     */
    public function calculatePrice(Request $request, PricingEngine $pricing): JsonResponse
    {
        $price = $pricing->calculate(
            Product::findOrFail($request->input('product_id')),
            $request->input('quantity'),
            $request->input('options', []),
        );

        return response()->json(['price' => $price]);
    }
}
```

## 4.5 Eloquent 数据层设计模式

Eloquent 是 Laravel 的 ActiveRecord ORM，与 MyBatis / JPA 的核心差异在于：模型类自身即数据访问层，无需独立的 Mapper / DAO / Repository（除非项目规模确实需要）。Eloquent 提供了丰富的内置机制处理数据转换、查询复用、关联加载、事件监听。

### 4.5.1 Accessors / Mutators / Casts

**Eloquent Accessors/Mutators**（Attribute::make，兼容 PHP 8.1+）：

**自定义 Cast（金额对象）**：

```php
<?php

namespace App\Domains\Common\Castables;

use App\Domains\Common\ValueObjects\Money;
use Illuminate\Contracts\Database\Eloquent\CastsAttributes;
use Illuminate\Database\Eloquent\Model;

class MoneyCast implements CastsAttributes
{
    public function get(Model $model, string $key, mixed $value, array $attributes): Money
    {
        return new Money((int) $value);
    }

    public function set(Model $model, string $key, mixed $value, array $attributes): int
    {
        return $value instanceof Money ? $value->amount : (int) ($value * 100);
    }
}
```

### 4.5.4 Observers

Observer 用于监听模型生命周期事件（creating/created/updating/updated/deleting/deleted），实现横切关注点（如审计日志、缓存清除、事件广播）：

```php
<?php

namespace App\Domains\Order\Observers;

use App\Domains\Order\Events\OrderCreated;
use App\Domains\Order\Events\OrderStatusChanged;
use App\Domains\Order\Models\Order;

class OrderObserver
{
    /**
     * 订单创建后：发送领域事件、清除缓存
     */
    public function created(Order $order): void
    {
        event(new OrderCreated($order));

        // 清除客户订单列表缓存
        cache()->forget("customer:{$order->customer_id}:orders");
    }

    /**
     * 状态更新时：校验状态机、记录变更日志
     */
    public function updating(Order $order): void
    {
        if ($order->isDirty('status')) {
            $oldStatus = OrderStatus::from($order->getOriginal('status'));
            $newStatus = $order->status;

            throw_unless(
                $oldStatus->canTransitionTo($newStatus),
                InvalidStateTransitionException::class,
                "Cannot transition from {$oldStatus->label()} to {$newStatus->label()}"
            );
        }
    }

    /**
     * 订单更新后：广播状态变更
     */
    public function updated(Order $order): void
    {
        if ($order->wasChanged('status')) {
            $oldStatus = OrderStatus::from($order->getOriginal('status'));
            event(new OrderStatusChanged($order, $oldStatus, $order->status));
        }
    }

    /**
     * 订单删除后：级联清理
     */
    public function deleted(Order $order): void
    {
        // 软删除时清理关联缓存
        cache()->tags(["order:{$order->id}"])->flush();
    }
}
```

## 4.6 Action 类模式与业务组织

在 Laravel 生态中，**Action 类（单一职责业务动作类）**是替代 Java `ServiceImpl` 的最佳实践。每个 Action 只做一件事，命名即意图，天然符合单一职责原则。

### 4.6.1 Action 类设计规范

```php
<?php

namespace App\Domains\Order\Actions;

use App\Domains\Common\Enums\OrderStatus;
use App\Domains\Order\Events\OrderCreated;
use App\Domains\Order\Models\Order;
use App\Domains\Payment\Actions\CreatePendingPaymentAction;
use App\Domains\Product\Services\InventoryService;
use App\Services\Pricing\PricingEngine;
use Illuminate\Support\Facades\DB;

class CreateOrderAction
{
    public function __construct(
        private readonly PricingEngine $pricing,
        private readonly InventoryService $inventory,
        private readonly CreatePendingPaymentAction $createPayment,
    ) {}

    /**
     * 执行下单动作
     */
    public function execute(array $data): Order
    {
        return DB::transaction(function () use ($data) {
            // 1. 创建订单主表
            $order = Order::create([
                'customer_id' => $data['customer_id'],
                'status' => OrderStatus::REGISTERED,
                'out_status_name' => OrderOutStatus::PENDING_PAYMENT,
                'factory_code' => $data['factory_code'],
                'delivery_type' => $data['delivery_type'],
                'order_sign' => 0,
            ]);

            // 2. 创建订单明细
            $order->items()->createMany($data['items']);

            // 3. 计价引擎计算价格
            $this->pricing->apply($order);

            // 4. 锁定库存
            $this->inventory->lock($order);

            // 5. 创建待支付流水（通过事件解耦或直接调用）
            $this->createPayment->execute($order);

            // 6. 触发领域事件
            event(new OrderCreated($order));

            return $order;
        });
    }
}
```

### 4.6.2 Action 与 Pipeline 组合

复杂业务流程（如计价引擎）使用 Laravel Pipeline 将多个步骤串联：

```php
<?php

namespace App\Services\Pricing;

use App\Domains\Order\Models\Order;
use Illuminate\Pipeline\Pipeline;

/** @deprecated FIX-P0: 统一使用 PricingPipeline，本类将在 v2.8 中移除 */
class PricingEngine
{
    /**
     * 计价管道步骤
     */
    private array $pipes = [
        CalculateBasePrice::class,      // 基础价格
        ApplyQuantityDiscount::class,   // 数量折扣
        ApplyVipDiscount::class,        // VIP 折扣
        ApplyCouponDiscount::class,     // 优惠券抵扣
        CalculateExpressFee::class,     // 运费计算
        CalculatePackagingFee::class,   // 包装费
        ApplyDepositRule::class,        // 定金规则
    ];

    public function apply(Order $order): void
    {
        app(Pipeline::class)
            ->send($order)
            ->through($this->pipes)
            ->thenReturn();

        $order->save();
    }
}
```

每个管道步骤：

```php
<?php

namespace App\Services\Pricing\Pipes;

use App\Domains\Order\Models\Order;
use Closure;

class CalculateBasePrice
{
    public function handle(Order $order, Closure $next): Order
    {
        $basePrice = $order->items->sum(
            fn ($item) => $item->unit_price * $item->quantity
        );

        $order->product_sum = $basePrice;

        return $next($order);
    }
}
```

### 4.6.3 Action vs Service 的选择

| 场景 | 推荐方式 | 说明 |
|------|---------|------|
| 单一业务动作（创建订单、审核订单） | **Action 类** | 一个类一个公共方法，命名精确 |
| 一组相关查询（CRUD、分页、过滤） | **Repository 类** | 封装数据访问逻辑，返回 Eloquent Builder 或 Collection |
| 跨领域协调（下单同时涉及库存/支付/通知） | **Action 组合** | 主 Action 调用子领域 Action，通过事件进一步解耦 |
| 工具类（金额计算、地址解析） | **静态方法 / 函数** | 无状态纯函数，无需实例化 |

> **禁止出现 `ServiceImpl`**：PHP 不需要接口默认分离。除非真的有多个实现（如支付网关），否则不要为 Service 创建接口。

### 4.7.1 FormRequest（替代 Java @Valid）

Laravel FormRequest 将验证逻辑从控制器中抽离，自动执行验证，失败时自动返回 422 JSON 响应。

```php
<?php

namespace App\Http\Requests;

use App\Domains\Common\Enums\DeliveryType;
use App\Domains\Common\Enums\FactoryCode;
use Illuminate\Foundation\Http\FormRequest;
use Illuminate\Validation\Rule;

class StoreOrderRequest extends FormRequest
{
    /**
     * 当前用户是否有权发起该请求
     */
    public function authorize(): bool
    {
        return $this->user()->can('create', Order::class);
    }

    /**
     * 验证规则
     */
    public function rules(): array
    {
        return [
            'factory_code' => ['required', Rule::enum(FactoryCode::class)],
            'delivery_type' => ['required', Rule::enum(DeliveryType::class)],
            'address_id' => ['required', 'exists:customer_addresses,id'],
            'items' => ['required', 'array', 'min:1'],
            'items.*.product_id' => ['required', 'exists:products,id'],
            'items.*.quantity' => ['required', 'integer', 'min:1'],
            'items.*.options' => ['sometimes', 'array'],
            'coupon_id' => ['sometimes', 'exists:coupons,id'],
            'remark' => ['sometimes', 'string', 'max:500'],
        ];
    }

    /**
     * 自定义错误消息
     */
    public function messages(): array
    {
        return [
            'items.required' => '订单至少需要包含一个商品',
            'items.*.product_id.exists' => '选择的商品不存在',
        ];
    }

    /**
     * 字段别名（前端字段名映射）
     */
    public function attributes(): array
    {
        return [
            'factory_code' => '生产工厂',
            'address_id' => '收货地址',
        ];
    }
}
```

## 4.8 服务提供者注册机制

ServiceProvider 是 Laravel 模块化的核心，负责**绑定服务到容器**、**注册事件监听**、**加载配置/路由/视图**。

### 4.8.1 内置 Provider 分工

| Provider | 职责 |
|---------|------|
| `AppServiceProvider` | 全局单例绑定、宏定义、模型观察器注册 |
| `AuthServiceProvider` | Gate / Policy 授权策略注册 |
| `EventServiceProvider` | Event → Listener 映射、模型 Observer 注册 |
| `RouteServiceProvider` | 路由组配置、RateLimiter 定义、路由模型绑定 |
| `DomainServiceProvider`（自定义） | 领域服务绑定、接口实现注入 |

### 4.8.2 自定义 DomainServiceProvider

```php
<?php

namespace App\Providers;

use App\Domains\Payment\Contracts\PaymentGateway;
use App\Domains\Payment\Gateways\AlipayGateway;
use App\Domains\Payment\Gateways\WechatPayGateway;
use App\Services\Pricing\PricingEngine;
use Illuminate\Support\ServiceProvider;

class DomainServiceProvider extends ServiceProvider
{
    public function register(): void
    {
        // 支付网关：根据配置动态绑定
        $this->app->bind(PaymentGateway::class, function ($app) {
            return match (config('payment.default')) {
                'wechat' => $app->make(WechatPayGateway::class),
                'alipay' => $app->make(AlipayGateway::class),
            };
        });

        // 计价引擎：单例
        $this->app->singleton(PricingEngine::class);

        // 仓储类绑定（若使用 Repository 模式）
        $this->app->bind(
            \App\Domains\Order\Contracts\OrderRepository::class,
            \App\Domains\Order\Repositories\EloquentOrderRepository::class
        );
    }

    public function boot(): void
    {
        // 注册模型观察者
        \App\Domains\Order\Models\Order::observe(
            \App\Domains\Order\Observers\OrderObserver::class
        );

        // 注册自定义验证规则
        \Illuminate\Support\Facades\Validator::extend('phone', function ($attribute, $value) {
            return preg_match('/^1[3-9]\d{9}$/', $value);
        });
    }
}
```

### 4.8.3 路由模型绑定

RouteServiceProvider 中定义隐式模型绑定，将路由参数自动解析为模型实例：

```php
<?php

namespace App\Providers;

use App\Domains\Order\Models\Order;
use Illuminate\Cache\RateLimiting\Limit;
use Illuminate\Http\Request;
use Illuminate\Support\Facades\RateLimiter;

class RouteServiceProvider extends ServiceProvider
{
    public function boot(): void
    {
        // API 限流：每分钟 60 次

        Route::bind('order', function (string $value) {
            return Order::where('order_sn', $value)
                        ->orWhere('id', $value)
                        ->firstOrFail();
        });

        // 路由组配置
        $this->routes(function () {
            Route::middleware('api')
                ->prefix('api/v1')
                ->group(base_path('routes/api.php'));

            Route::middleware('api')
                ->prefix('api/admin')
                ->group(base_path('routes/api_admin.php'));
        });
    }
}
```

## 4.9 与 Java 版分层设计的核心差异总结

| 维度 | Java 版（Spring Boot） | PHP/Laravel 版 |
|------|----------------------|---------------|
| 服务组织 | Maven 多模块 / 微服务 | **模块化单体**：`app/Domains/` 单一代码库 |
| 数据层 | Entity + Mapper XML + DAO + ServiceImpl | **Eloquent Model**：富模型，无 XML Mapper |
| 验证 | `@Valid` + `BindingResult` | **FormRequest**：自动验证，失败自动 422 |
| 响应封装 | DTO + VO + `Result<T>` | **API Resource**：灵活转换，无需严格分层 |
| 依赖注入 | `@Autowired` + `@Component` | **构造函数注入**：容器自动解析，零注解 |
| 事件驱动 | Spring EventBus | **Laravel Event/Listener**：同步/异步/广播 |
| 异步处理 | `@Async` + ThreadPool | **Queue Job**：Redis 驱动，Horizon 监控 |
| 连接池 | HikariCP / Druid | **Octane 连接复用**：常驻内存，无需手动配置 |
| 定时任务 | XXL-JOB | **Laravel Schedule**：内置 cron + Redis 分布式锁 |
| 枚举 | Java Enum | **PHP Backed Enum**：`match` 表达式、`label()` 方法 |

#### OrderCreateSource (lM)

订单创建来源位掩码。一个订单可同时具备多个来源特征，使用按位或运算组合。

```php
<?php

declare(strict_types=1);

namespace App\Enums;

/**
 * 创建来源位掩码
 * PRD键名: lM
 */
enum OrderCreateSource: int
{
    case ONLINE_DESIGN     = 1;    // 在线设计
    case CUSTOM_MATERIAL   = 2;    // 自定义材料
    case MEITU_CLOUD       = 4;    // 美图云修
    case FAVORITE_QUOTE    = 8;    // 收藏报价
    case YIYIN             = 16;   // 艺印
    case LEAD_FOLLOW       = 32;   // 线索跟进
    case INK_COVERAGE      = 64;   // 油墨覆盖率
    case SAME_DAY_DELIVERY = 128;  // 当日达

    public function label(): string
    {
        return match ($this) {
            self::ONLINE_DESIGN     => '在线设计',
            self::CUSTOM_MATERIAL   => '自定义材料',
            self::MEITU_CLOUD       => '美图云修',
            self::FAVORITE_QUOTE    => '收藏报价',
            self::YIYIN             => '艺印',
            self::LEAD_FOLLOW       => '线索跟进',
            self::INK_COVERAGE      => '油墨覆盖率',
            self::SAME_DAY_DELIVERY => '当日达',
        };
    }

    /**
     * 检测位掩码中是否包含当前标志
     */
    public function hasFlag(int $flags): bool
    {
        return ($flags & $this->value) === $this->value;
    }

    /**
     * 将当前标志添加到位掩码
     */
    public function addFlag(int $flags): int
    {
        return $flags | $this->value;
    }

    /**
     * 从位掩码中移除当前标志
     */
    public function removeFlag(int $flags): int
    {
        return $flags & ~$this->value;
    }

    /**
     * 将整型位掩码解析为枚举数组
     */
    public static function decode(int $flags): array
    {
        return array_filter(self::cases(), fn(self $c) => $c->hasFlag($flags));
    }

    /**
     * 将枚举数组编码为整型位掩码
     */
    public static function encode(array $cases): int
    {
        return array_reduce($cases, fn(int $carry, self $c) => $carry | $c->value, 0);
    }
}
```

**枚举系统总结：**

1. 以上26个枚举与SDD第4.3节已定义的16个枚举（`OrderStatus`、`OrderOutStatus`、`AfterSaleStatus`、`OrderModifyStatus`、`AuthStatus`、`PayType`、`SubPermission`、`DeliveryType`、`CommentLevel`、`PointType`、`TicketStatus`、`OrderSign`、`InvoiceStatus`、`InvoiceBusinessType`、`VipLevel`）共同构成 **42套完整枚举系统**，实现PRD v3.0 全量业务语义覆盖。

2. 位掩码枚举 `OrderCreateSource` 提供 `hasFlag()` / `addFlag()` / `removeFlag()` / `decode()` / `encode()` 全套位运算工具方法，便于在订单模型中进行来源特征的组合查询与存储。

3. 状态机枚举（`BrandAuditStatus`、`ProductionScheduleStatus`、`HelpFaqStatus`、`KnowledgeArticleStatus`、`ProductionWorkOrderStatus`、`AfterSaleFlowStatus`、`PlatformOrderStatus`）均提供 `canTransitionTo()` 与 `isTerminal()` 方法，与已定义的 `OrderStatus`、`AfterSaleStatus` 保持同一套状态机校验范式，可直接在 Eloquent Observer 或领域服务中调用。

4. 所有枚举均声明 `declare(strict_types=1)`，使用 `int` 作为底层类型（与数据库 `tinyint` / `smallint` 严格对应），命名空间统一为 `App\Enums`，满足 Laravel 10+ 的 Eloquent `$casts` 自动转换要求。
