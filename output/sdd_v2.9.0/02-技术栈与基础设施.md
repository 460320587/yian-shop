**Octane 常驻内存模式说明**：

传统 PHP-FPM 每个请求都需重新启动框架、重新解析依赖、重新建立数据库连接。Laravel Octane 将应用启动后驻留内存，Worker 进程循环处理请求：

- **启动阶段**（一次）：加载 Composer 自动加载器、注册 ServiceProvider、建立数据库/Redis 连接池。
- **请求阶段**（循环）：仅执行路由匹配 → 中间件 → 控制器 → 响应，框架本身零重复加载。
- **连接复用**：Octane 自动保持 PDO 与 Redis 连接在 Worker 生命周期内复用，无需手动维护连接池配置。
- **协程支持**（Swoole 模式）：在单个 Worker 内可通过 `Swoole\Coroutine` 并发发起多个 IO 请求（如同时查询第三方物流 API）。

```php
// config/octane.php
return [
    'server' => env('OCTANE_SERVER', 'swoole'),
    'workers' => env('OCTANE_WORKERS', 4),
    'task_workers' => env('OCTANE_TASK_WORKERS', 2),
    'max_requests' => 500, // 每个 Worker 处理 500 次请求后优雅重启，防止内存泄漏
    'rr_binary' => env('OCTANE_RR_BINARY', base_path('rr')),
];
```

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| `workers` | 4 × CPU 核心数 | 根据并发量调整，IO 密集型可适度增加 |
| `task_workers` | 2 × CPU 核心数 | 专门处理耗时任务（如图片压缩、报表生成） |
| `max_requests` | 500 | Worker 处理 500 次请求后自动重启，防止内存泄漏 |

> **⚠️ 注意**：PHP-FPM 无 Java 式连接池概念（如 HikariCP）。Octane 的常驻内存特性天然实现了**连接复用**——PDO 连接在 Worker 启动时建立，请求间复用同一连接。若使用传统 PHP-FPM，则依赖 PDO 的短连接 + 数据库端 `wait_timeout` 调优即可。

**为什么不使用 RabbitMQ 原生客户端？**

Laravel Queue 的 Redis 驱动在生产环境中完全足够：

- **可靠性**：Horizon 监控所有 Job，失败自动重试（指数退避），死信队列（Failed Jobs）持久化到数据库。
- **优先级**：支持多个队列（`high`, `default`, `low`），紧急通知走 `high`，报表生成走 `low`。
- **批量处理**：Horizon 支持批量消费，减少 Redis 往返。
- **运维成本**：无需维护 RabbitMQ 集群，Redis 本身已是缓存基础设施。

```php
// config/queue.php
'redis' => [
    'driver' => 'redis',
    'connection' => 'default',
    'queue' => 'default',
    'retry_after' => 90, // Job 执行超过 90 秒视为失败，重新投递
    'block_for' => null,
    'after_commit' => true, // 数据库事务提交后才入队，防止数据不一致
],
```

## 3.5 缓存防护策略

Laravel Cache 提供统一的缓存抽象，底层对接 Redis。本节基于 Laravel Cache + Redis 实现穿透、击穿、雪崩三层防护。

### 3.5.1 缓存穿透防护

**问题**：大量查询数据库中不存在的 Key（如伪造订单号），导致每次请求都打到数据库。

**方案**：布隆过滤器（Bloom Filter）预拦截 + 空值缓存兜底。

```php
<?php

namespace App\Cache;

use Illuminate\Support\Facades\Cache;
use Illuminate\Support\Facades\Redis;

class PenetrationGuard
{
    private const EMPTY_TTL = 60; // 空值缓存 60 秒

    public function __construct(
        private BloomFilter $bloomFilter
    ) {}

    public function get(string $key, callable $dbLoader): mixed
    {
        // 1. 布隆过滤器快速拦截：必定不存在的 Key 直接返回 null
        if (!$this->bloomFilter->mayExist($key)) {
            return null;
        }

        // 2. 查 Laravel Cache（底层 Redis）
        $value = Cache::get($key);
        if ($value === '__EMPTY__') {
            return null; // 空值缓存命中
        }
        if ($value !== null) {
            return $value;
        }

        // 3. 回源查询
        $value = $dbLoader();

        if ($value === null) {
            // 空值缓存：防止同一不存在的 Key 反复查 DB
            Cache::put($key, '__EMPTY__', self::EMPTY_TTL);
        } else {
            Cache::put($key, $value, $this->randomTtl(300));
        }

        return $value;
    }

    private function randomTtl(int $base): int
    {
        // ±10% 随机偏移，防止批量同时过期
        $offset = (mt_rand() / mt_getrandmax() - 0.5) * 0.2;
        return max(1, (int) ($base * (1 + $offset)));
    }
}
```

**布隆过滤器实现**（基于 Redis Bitmap）：

```php
<?php

namespace App\Cache;

use Illuminate\Support\Facades\Redis;

class BloomFilter
{
    private string $key = 'bloom:orders';
    private int $size = 1_000_000; // Bitmap 位数
    private int $hashCount = 5;    // 哈希函数个数

    public function add(string $element): void
    {
        foreach ($this->hashes($element) as $offset) {
            Redis::setbit($this->key, $offset, 1);
        }
    }

    public function mayExist(string $element): bool
    {
        foreach ($this->hashes($element) as $offset) {
            if (Redis::getbit($this->key, $offset) === 0) {
                return false;
            }
        }
        return true;
    }

    private function hashes(string $element): array
    {
        $hashes = [];
        for ($i = 0; $i < $this->hashCount; $i++) {
            $hashes[] = crc32($element . ':' . $i) % $this->size;
        }
        return $hashes;
    }
}
```

### 3.5.2 缓存击穿防护

**问题**：热点 Key（如首页热销商品）突然过期，瞬间大量请求同时打到数据库。

**方案**：互斥锁（Redis `SET NX EX`）+ 单进程回源，其他进程等待或返回降级数据。

```php
<?php

namespace App\Cache;

use Illuminate\Support\Facades\Cache;
use Illuminate\Support\Facades\Redis;

class BreakdownGuard
{
    public function get(string $key, callable $dbLoader, int $ttlSeconds): mixed
    {
        // 1. 先查缓存
        $value = Cache::get($key);
        if ($value !== null) {
            return $value;
        }

        $lockKey = 'lock:cache:' . $key;
        $lockValue = uniqid('', true);

        // 2. 尝试获取分布式锁（5 秒自动释放，防止死锁）
        $locked = Redis::set($lockKey, $lockValue, 'NX', 'EX', 5);

        if (!$locked) {
            // 未获取到锁，等待 50ms 后重试读缓存
            usleep(50_000);
            return Cache::get($key);
        }

        try {
            // 3. 双重检查：获取锁后可能已有其他进程回填缓存
            $value = Cache::get($key);
            if ($value !== null) {
                return $value;
            }

            // 4. 回源查询并回填
            $value = $dbLoader();
            if ($value !== null) {
                Cache::put($key, $value, $this->randomTtl($ttlSeconds));
            }
            return $value;
        } finally {
            // 5. 安全释放锁：Lua 脚本保证原子性（只释放自己持有的锁）
            Redis::eval(
                'if redis.call("get", KEYS[1]) == ARGV[1] then return redis.call("del", KEYS[1]) else return 0 end',
                1, $lockKey, $lockValue
            );
        }
    }

    private function randomTtl(int $base): int
    {
        $offset = (mt_rand() / mt_getrandmax() - 0.5) * 0.2;
        return max(1, (int) ($base * (1 + $offset)));
    }
}
```

### 3.5.3 缓存雪崩防护

**问题**：大量缓存 Key 在同一时刻过期，或 Redis 节点故障，导致数据库瞬间被打垮。

**方案**：TTL 随机偏移 + 多级缓存（Laravel Array Store L1 + Redis L2）+ Redis 故障降级。

```php
<?php

namespace App\Cache;

use Illuminate\Support\Facades\Cache;

class SnowslideGuard
{
    /**
     * L1: 进程内数组缓存（30 秒），防止同一进程内反复读 Redis
     */
    private array $local = [];
    private array $expiry = [];
    private int $localTtl = 30;

    public function get(string $key, callable $dbLoader, int $redisTtl): mixed
    {
        $now = time();

        // L1: 进程内缓存（仅适用于 Octane 常驻内存模式，传统 PHP-FPM 每次请求独立进程）
        if (isset($this->local[$key]) && ($this->expiry[$key] ?? 0) > $now) {
            return $this->local[$key];
        }

        // L2: Redis 分布式缓存
        try {
            $cached = Cache::store('redis')->get($key);
            if ($cached !== null) {
                $this->local[$key] = $cached;
                $this->expiry[$key] = $now + $this->localTtl;
                return $cached;
            }
        } catch (\Exception) {
            // Redis 故障，跳过 L2，直接进入 L3 回源
        }

        // L3: 数据库回源
        $value = $dbLoader();
        if ($value !== null) {
            $ttl = $this->randomTtl($redisTtl);
            try {
                Cache::store('redis')->put($key, $value, $ttl);
            } catch (\Exception) {
                // Redis 故障，仅返回 DB 值，不报错
            }
        }
        return $value;
    }

    public static function randomTtl(int $base): int
    {
        $offset = (mt_rand() / mt_getrandmax() - 0.5) * 0.2; // ±10%
        return max(1, (int) ($base * (1 + $offset)));
    }
}
```

### 3.5.4 Redis 故障降级

当 Redis 不可用时，直接查数据库并通过 Laravel RateLimiter 限流保护：

```php
<?php

namespace App\Cache;

use Illuminate\Support\Facades\Cache;
use Illuminate\Support\Facades\Log;
use Illuminate\Support\Facades\RateLimiter;

class RedisFallback
{
    public function execute(string $key, callable $cacheLoader, callable $dbLoader): mixed
    {
        try {
            $value = Cache::store('redis')->get($key);
            if ($value !== null) {
                return $value;
            }

            // 缓存未命中，正常回源
            $value = $dbLoader();
            if ($value !== null) {
                Cache::store('redis')->put($key, $value, 300);
            }
            return $value;
        } catch (\Exception $e) {
            Log::warning('Redis unavailable, fallback to DB', ['key' => $key, 'error' => $e->getMessage()]);

            // 降级：限流保护 DB，每 Key 每秒最多 10 次回源
            return RateLimiter::attempt('db-fallback:' . $key, 10, function () use ($dbLoader) {
                return $dbLoader();
            });
        }
    }
}
```

| 防护场景 | 策略 | Laravel 实现要点 |
|----------|------|-----------------|
| 缓存穿透 | 布隆过滤器 + 空值缓存 | `Redis::setbit/getbit` 实现 Bloom Filter；`Cache::put($key, '__EMPTY__', 60)` 兜底 |
| 缓存击穿 | 互斥锁 + 单进程回源 | `Redis::set('NX', 'EX', 5)` 分布式锁；双重检查 + Lua 释放锁 |
| 缓存雪崩 | TTL 随机偏移 + 多级缓存 | `randomTtl(300)` ±10%；Octane Array Store (L1) + Redis (L2) |
| Redis 故障 | 降级 + DB 限流 | `RateLimiter::attempt()` 滑动窗口限流，防止 DB 被压垮 |

## 3.6 定时任务调度

| 维度 | Java 版方案 | Laravel 版方案 |
|------|------------|---------------|
| 调度引擎 | XXL-JOB 分布式调度 | **Laravel Schedule + Artisan Command** |
| 分布式锁 | XXL-JOB 内置 | **Redis 分布式锁** |
| 任务定义 | Java 类 + @XxlJob 注解 | **Artisan Command 类 + Schedule 闭包** |
| 监控 | XXL-JOB 控制台 | **Horizon / Pulse + 日志** |

Laravel Schedule 在 `routes/console.php` 中统一定义：

```php
<?php

use Illuminate\Support\Facades\Schedule;
use App\Jobs\SyncBestsellerRank;
use App\Console\Commands\ClearExpiredCarts;

// 每日凌晨 2:00 刷新热销榜
Schedule::job(new SyncBestsellerRank)->dailyAt('02:00');

// 每 5 分钟清理过期购物车（分布式锁防止多实例重复执行）
Schedule::command(ClearExpiredCarts::class)
    ->everyFiveMinutes()
    ->onOneServer(); // Redis 分布式锁保证仅单实例执行

// 每 5 分钟取消超时未支付订单（24小时阈值）
Schedule::command('order:cancel-overdue')
    ->everyFiveMinutes()
    ->onOneServer()
    ->withoutOverlapping(10);

// 每小时生成生产排期报表
Schedule::command('report:production-schedule')
    ->hourly()
    ->onOneServer()
    ->withoutOverlapping(30); // 30 分钟内未执行完则跳过
```

**分布式锁实现**（`onOneServer()` 底层）：

Laravel 原生支持基于 Redis 的分布式锁，`onOneServer()` 自动在多个 Octane 实例间协调，确保同一时刻只有一个实例执行该任务。

```php
// config/schedule.php
return [
    'cache' => [
        'store' => 'redis',
        'prefix' => 'laravel_schedule',
    ],
];
```

> **⚠️ 注意**：严禁使用 XXL-JOB。Laravel Schedule + Redis 分布式锁完全满足本项目调度需求，无需引入额外 Java 基础设施。
