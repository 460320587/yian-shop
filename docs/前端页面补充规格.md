## 1. 登录页 `/login` (PC+H5)

### 1.1 页面布局

**PC端**
```
┌─────────────────────────────────────────────┐
│  左侧: 品牌展示区 (占50%)                      │
│  - 全屏背景图 (印刷厂/成品展示)                │
│  - Logo + Slogan "专业商务印刷，品质触手可及"   │
│  - 底部: 版权信息 + ICP备案                    │
├─────────────────────────────────────────────┤
│  右侧: 登录表单区 (占50%, max-width 420px)    │
│  - Tab切换: 账号登录 / 短信登录                │
│  - 手机号输入框                               │
│  - 密码输入框 (账号登录) / 验证码输入框(短信)   │
│  - 图形验证码 (Canvas 120x40px)               │
│  - 登录按钮 (primary, 全宽)                   │
│  - 快捷登录: 微信扫码 / QQ授权                 │
│  - 底部链接: 忘记密码 | 立即注册               │
└─────────────────────────────────────────────┘
```

**H5端**
```
┌─────────────────────┐
│  Logo居中 (120x40)   │
│  欢迎回来            │
│  手机号输入框        │
│  密码/验证码输入框   │
│  图形验证码          │
│  登录按钮            │
│  快捷登录区          │
│  忘记密码 | 注册     │
└─────────────────────┘
```

### 1.2 字段与交互

| 字段 | 类型 | 必填 | 校验规则 | 错误提示 |
|------|:----:|:----:|----------|----------|
| 手机号 | Input | ✅ | 1开头11位数字正则 | 请输入正确的手机号 |
| 密码 | Password | ✅ (账号登录) | 8-20位，含字母+数字+特殊字符 | 密码格式不正确 |
| 短信验证码 | Input | ✅ (短信登录) | 6位数字 | 验证码错误 |
| 图形验证码 | Input | ✅ | 4位字母数字 | 验证码错误 |

**交互逻辑**：
- 账号Tab: 手机号+密码+图形验证码 → POST `/api/v1/auth/login`
- 短信Tab: 手机号+短信验证码+图形验证码 → POST `/api/v1/auth/login-sms`
- 图形验证码点击刷新: GET `/api/v1/captcha?key={random}`
- 短信验证码发送: 60秒倒计时，限流5次/小时
- 登录成功: Sanctum SPA模式自动写入httpOnly Cookie，跳转`redirect`参数或首页
- 登录失败>3次: 要求图形验证码
- 登录失败>5次: 账号锁定30分钟

### 1.3 数据来源

| 数据 | API | 说明 |
|------|-----|------|
| 登录 | POST `/api/v1/auth/login` | 返回token+user信息 |
| 短信验证码 | POST `/api/v1/auth/sms-code` | 发送短信 |
| 图形验证码 | GET `/api/v1/captcha` | 返回图片+key |

│  - 短信验证码 (60s倒计时)                     │
│  - 密码设置 (强度实时检测: 弱/中/强)           │
│  - 确认密码                                   │
│  - 用户协议勾选框                             │
│  - 注册按钮                                   │
│  - 已有账号？立即登录                         │
└─────────────────────────────────────────────┘
```

### 2.2 字段与交互

| 字段 | 类型 | 必填 | 校验规则 | 错误提示 |
|------|:----:|:----:|----------|----------|
| 手机号 | Input | ✅ | 1开头11位，唯一性校验 | 手机号已注册/格式错误 |
| 图形验证码 | Input | ✅ | 4位 | 验证码错误 |
| 短信验证码 | Input | ✅ | 6位 | 验证码错误/已过期 |
| 密码 | Password | ✅ | 8-20位，字母+数字+特殊字符 | 密码强度不足 |
| 确认密码 | Password | ✅ | 与密码一致 | 两次密码不一致 |
| 用户协议 | Checkbox | ✅ | 必须勾选 | 请阅读并同意协议 |

**交互逻辑**：
- 手机号失焦: 调用 GET `/api/v1/auth/check-phone?phone=xxx` 校验唯一性
- 密码强度: 实时检测，显示色条（红/黄/绿）
- 注册成功: 自动登录，跳转首页

│  │  操作按钮                                │  │
│  │  [取消订单] [返回修改] [确认支付]        │  │
│  └───────────────────────────────────────┘  │

```

**H5端**
```
┌─────────────────────┐
│  返回 | 订单支付     │
│  应付金额 ¥4,620.00  │
│  倒计时 23:59:59     │
│  ─────────────────  │
│  支付方式            │
│  ┌───────────────┐  │
│  │ 微信支付   ▶  │  │
│  │ 支付宝     ▶  │  │
│  │ 余额支付   ▶  │  │
│  └───────────────┘  │
│  [确认支付]          │
└─────────────────────┘
```

### 4.2 字段与交互

| 字段 | 类型 | 说明 |
|------|------|------|
| 订单号 | Text | 只读 |
| 应付金额 | Text | 只读，红色大字号 |
| 倒计时 | Timer | 订单创建后24小时，每秒更新 |
| 支付方式 | Radio | 微信/支付宝/余额/对公转账 |

**交互逻辑**：
- 页面加载: GET `/api/v1/orders/{order_no}` 获取订单信息
- 选择微信支付: POST `/api/v1/payments` (type=wechat) → 返回二维码 → 轮询支付状态
- 选择支付宝: POST `/api/v1/payments` (type=alipay) → 返回二维码 → 轮询支付状态
- 选择余额: POST `/api/v1/payments` (type=wallet) → 输入支付密码 → 直接扣款
- 轮询逻辑: 每3秒调用 GET `/api/v1/payments/{id}`，最多200次（600秒）
- 支付成功: 跳转 `/order/{order_no}` 显示"支付成功"
- 支付失败/超时: 提示"支付失败"，提供"重新支付"按钮
- 取消订单: 调用 POST `/api/v1/orders/{order_no}/cancel`

**二维码展示**：
- PC端: 直接显示二维码图片，用户扫码
- H5端: 调起微信H5支付/支付宝WAP支付（跳转外部链接）

### 4.3 数据来源

| 数据 | API | 说明 |
|------|-----|------|
| 订单信息 | GET `/api/v1/orders/{order_no}` | 订单详情 |
| 创建支付 | POST `/api/v1/payments` | 创建支付单 |
| 查询支付 | GET `/api/v1/payments/{id}` | 轮询状态 |
| 余额查询 | GET `/api/v1/wallet/balance` | 显示余额 |
