## 1. 通用布局规范

### 1.1 PC端布局

```
┌─────────────────────────────────────────────┐
│  Header (固定高度64px)                       │
│  Logo | 搜索框 | 分类导航 | 消息 | 购物车 | 个人中心  │
├─────────────────────────────────────────────┤
│                                             │
│           Main Content Area                 │
│           (最小宽度1200px, 居中)              │
│                                             │
├─────────────────────────────────────────────┤
│  Footer (固定高度200px)                      │
│  关于我们 | 帮助中心 | 联系方式 | 备案信息     │
└─────────────────────────────────────────────┘
```

**Header组件规范**:
- Logo: 左侧，高度40px，点击跳转首页
- 搜索框: 居中，宽度400px，placeholder="搜索画册、名片、海报..."
- 分类导航: 搜索框右侧，"全部分类"下拉菜单（二级分类树）
- 右侧工具栏: 消息(Bell图标+未读红点) | 购物车(图标+数量角标) | 个人中心(头像/登录按钮)
- 滚动行为: 页面滚动>100px后Header添加`box-shadow: 0 2px 8px rgba(0,0,0,0.08)`

### 1.2 H5移动端布局

```
┌─────────────────────┐
│  Header (固定48px)   │
│  返回 | 标题 | 操作   │
├─────────────────────┤
│                     │
│   Main Content      │
│   (底部预留Tab高度)  │
│                     │
├─────────────────────┤
│  Bottom Tab (固定56px)│
│  首页 | 分类 | 购物车 | 我的 │
└─────────────────────┘
```

**H5 Tab Bar**:
- 首页: `home-o` / `home`
- 分类: `apps-o` / `apps`
- 购物车: `cart-o` / `cart` (角标显示未选中商品数)
- 我的: `user-o` / `user`

### P-001: 首页 (Home)

**路由**: `/`  
**布局**: Header + 内容区 + Footer (PC) / Header + 内容区 + TabBar (H5)  
**API**: `GET /api/v1/banners`, `GET /api/v1/categories`, `GET /api/v1/products?is_recommended=1`

#### 区块结构

| # | 区块名 | 高度/尺寸 | 组件 | 数据来源 | 交互 |
|---|--------|----------|------|---------|------|
| 1 | Banner轮播 | PC: 400px / H5: 180px | ElCarousel / VantSwipe | banners API | 点击跳转链接，5s自动切换 |
| 2 | 快捷分类入口 | PC: 120px / H5: 90px | 图标+文字Grid | categories API (前8个) | 点击跳转分类页 |
| 3 | 推荐商品 | 自适应 | AppCard网格 | products API | 点击跳转商品详情 |
| 4 | 新品上市 | 自适应 | AppCard横向滚动(H5) | products API (sort=newest) | 点击跳转商品详情 |
| 5 | 热销排行 | 自适应 | AppTable排名列表 | products API (sort=sales_desc) | 点击跳转商品详情 |
| 6 | 企业优势 | PC: 200px / H5: 隐藏 | 静态图文 | 静态配置 | — |

#### Banner数据结构

```
{
  id: int,
  image: string (URL),
  link_type: "product" | "category" | "url",
  link_target: string (product_id / category_id / URL),
  sort: int,
  display_period: [start_date, end_date]
}
```

#### 快捷分类入口 (H5)
- 2行4列Grid
- 图标: 分类icon (60×60px圆角)
- 文字: 分类名 (12px, 单行截断)

### P-002: 商品列表页 (Product List)

**路由**: `/category/:id` 或 `/search?keyword=xxx`  
**布局**: Header + 筛选栏 + 商品网格 + 分页  
**API**: `GET /api/v1/products`

#### 区块结构

| # | 区块名 | 组件 | 交互 |

- 评分: 黄色星星图标 + 分数 + 评价数
- 标签: ElTag组件，小尺寸

### P-004: 购物车页 (Cart)

**路由**: `/cart`  
**布局**: 商品列表 + 底部结算栏  
**API**: `GET /api/v1/cart`, `PUT /api/v1/cart/items/{id}`, `DELETE /api/v1/cart/items/{id}`

#### 购物车项卡片

```
┌─────────────────────────────────────────────┐
│ [✓]  [缩略图80×80]  商品名称              │
│       157g铜版纸 · 四色 · 48P · 覆膜      │
│       单价 ¥2.50  数量 [- 1000 +]  小计¥2500 │
│                                   [删除图标]  │
└─────────────────────────────────────────────┘
```

#### 字段规范

| 字段 | 组件 | 说明 |
|------|------|------|
| 选中状态 | ElCheckbox / VantCheckbox | 选中才计入结算 |
| 缩略图 | 80×80 WebP | 点击跳转商品详情 |
| 规格快照 | 文字 | pricing_snapshot各字段拼接 |
| 数量 | ElInputNumber / VantStepper | min=1, 变更时更新API |
| 小计 | 文字 | 实时计算: unit_price × quantity |
| 删除 | 图标按钮 | 确认弹窗后调用API-013 |

#### 失效商品
- 灰色背景，禁用勾选，显示"商品已下架/规格变更"标签
- 提供"删除失效商品"一键清除按钮

#### 底部结算栏 (H5固定底部，PC页面底部)

```
┌─────────────────────────────────────────────┐
│ [✓] 全选                   合计: ¥5,047.20  │
│                            已选2件            │
│                            [去结算 >]         │
└─────────────────────────────────────────────┘
```

- 结算按钮: 未选中时disabled，选中项有失效商品时提示"请移除失效商品"
- 点击结算: 跳转`/checkout?from=cart&item_ids=5001,5002`

│   - 抬头/税号  │               │               │

└───────────────┴───────────────┴───────────────┘

#### 收货信息区块

| 元素 | 组件 | 交互 |
|------|------|------|
| 地址列表 | 单选卡片 | 默认选中默认地址，点击展开选择其他地址 |
| 新增地址 | 按钮+弹窗 | 打开地址表单弹窗 |
| 编辑地址 | 图标 | 打开地址表单弹窗，预填充数据 |
| 配送范围提示 | 标签 | 地址超出配送范围时红色提示"该地址暂不支持配送" |

**地址卡片字段**:
- 联系人 + 手机号
- 完整地址 (省市区+详细地址)
- 标签 (公司/家/工厂) ElTag
- 默认标识

#### 发票信息区块

| 字段 | 组件 | 选项 | 默认 |
|------|------|------|------|
| 发票类型 | ElRadioGroup | 不开票 / 电子普票 / 电子专票 | 不开票 |
| 发票抬头 | ElInput | — | 企业认证名称(自动填充) |
| 纳税人识别号 | ElInput | 18位 | 自动填充 |
| 接收邮箱 | ElInput | 邮箱格式 | 自动填充 |

- 选择"不开票"时，其他字段隐藏
- 发票抬头自动从企业认证信息回填，可手动修改

#### 商品清单区块

- 每项显示: 缩略图 + 名称 + 规格快照 + 数量 + 小计
- 不可编辑数量（数量在购物车/商品详情确定）

#### 结算信息区块

| 字段 | 组件 | 说明 |
|------|------|------|
| 可用优惠券 | ElSelect | 下拉显示可用优惠券列表，含使用条件说明 |
| 积分抵扣 | ElSlider / 输入 | 当前积分{x}，可抵扣¥{x/100}，滑动选择抵扣数量 |
| 余额抵扣 | ElInputNumber | 可用余额{x}，输入抵扣金额，max=订单金额 |
| 商品总额 | 只读文字 | — |
| 运费 | 只读文字 | 根据地址实时计算，显示"¥0（包邮）"或具体金额 |
| 优惠 | 只读文字 | 负数，红色 |
| 应付总额 | 只读文字 | 24px加粗红色 |

#### 提交订单按钮

- 全宽按钮，文字"提交订单"
- disabled条件: 未选地址 / 地址超区 / 商品全部失效
- 点击: 调用API-015，`Idempotency-Key`使用UUIDv7
- 提交中: 按钮loading状态，防止重复提交
- 成功: 跳转支付页 `/payment/{payment_id}`
- 失败: Toast显示具体错误，停留在结算页

封装文件上传，支持直传签名、分片上传、图片预览、拖拽上传、格式/大小校验。

### 4.2 Props定义

```typescript
interface UploadFile {
  uid: string
  name: string
  size: number
  type: string
  status: 'ready' | 'uploading' | 'success' | 'error'
  percent?: number
  url?: string
  response?: any
  error?: string
}

interface AppUploadProps {
  // 基础
  modelValue: UploadFile[] | string | string[] // v-model
  action?: string                // 上传接口(有则直传，无则签名上传)
  
  // 文件限制
  accept?: string                // 接受类型，如 '.pdf,.jpg,.png' 或 'image/*'
  maxSize?: number               // 单文件最大大小(MB)
  maxCount?: number              // 最大文件数
  multiple?: boolean             // 多文件
  
  // 上传方式
  uploadType?: 'direct' | 'sign' | 'chunk' // direct:直传 / sign:签名 / chunk:分片
  signApi?: string               // 获取签名的API
  chunkSize?: number             // 分片大小(MB)，默认5
  
  // 图片特有
  isImage?: boolean              // 是否图片上传
  preview?: boolean              // 是否预览
  previewWidth?: number          // 预览图宽
  previewHeight?: number         // 预览图高
  crop?: boolean                 // 是否裁剪
  cropAspectRatio?: number       // 裁剪比例
  
  // 显示
  drag?: boolean                 // 拖拽上传
  listType?: 'text' | 'picture' | 'picture-card' // 列表样式
  showFileList?: boolean         // 显示文件列表
  buttonText?: string            // 按钮文字
  buttonIcon?: string            // 按钮图标
  tips?: string                  // 提示文字
  
  // 状态
  disabled?: boolean
  loading?: boolean
  
  // 自动处理
  autoUpload?: boolean           // 自动上传，默认true
  autoCompress?: boolean         // 自动压缩图片
  compressQuality?: number       // 压缩质量 0-1
  autoConvertWebp?: boolean      // 自动转WebP
}
```

### 4.3 Events

| 事件名 | 参数 | 说明 |
|--------|------|------|
| `update:modelValue` | `(files: UploadFile[])` | 文件列表变化 |
| `change` | `(file: UploadFile, fileList: UploadFile[])` | 文件状态变化 |
| `success` | `(file: UploadFile, response: any)` | 上传成功 |
| `error` | `(file: UploadFile, error: Error)` | 上传失败 |
| `progress` | `(file: UploadFile, percent: number)` | 上传进度 |
| `exceed` | `(files: File[], fileList: UploadFile[])` | 超出数量限制 |
| `remove` | `(file: UploadFile, fileList: UploadFile[])` | 移除文件 |
| `preview` | `(file: UploadFile)` | 预览文件 |

### 4.4 使用示例

```vue
<template>
  <AppUpload
    v-model="fileList"
    upload-type="sign"
    :sign-api="'/api/v1/upload/sign'"
    accept=".pdf"
    :max-size="500"
    :max-count="1"
    :multiple="false"
    button-text="上传印前文件"
    tips="支持PDF格式，最大500MB"
    @success="handleUploadSuccess"
    @error="handleUploadError"
  />
</template>
```
