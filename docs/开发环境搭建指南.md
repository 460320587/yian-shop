## 第一步：本地环境准备

### 1.1 安装必要软件

| 软件 | 版本 | 下载地址 |
|------|------|---------|
| Docker Desktop | 4.30+ | https://www.docker.com/products/docker-desktop |
| Git | 2.40+ | https://git-scm.com/download/win |
| VS Code | 最新 | https://code.visualstudio.com/ |
| Node.js | 22 LTS | https://nodejs.org/ |

**Docker Desktop 配置（重要）：**
1. 打开 Docker Desktop → Settings → General
2. ✅ 勾选 "Use the WSL 2 based engine"
3. Settings → Resources → WSL Integration
4. ✅ 开启与你的 WSL 发行版的集成
5. Settings → Resources → Memory：建议分配 **8GB**（你总共有 32GB，绰绰有余）

### 1.2 安装 VS Code 插件

```
- Docker (Microsoft)
- Dev Containers (Microsoft)
- PHP Intelephense (Ben Mewburn)
- Laravel Extension Pack (Winie)
- Vue - Official (Vue.js)
- ESLint (Microsoft)
- Prettier (Prettier)
- Thunder Client (rangav)  // API 测试，替代 Postman
```

### 1.3 配置 Git

```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
git config --global core.autocrlf false    # Windows 必备，避免换行符问题
git config --global init.defaultBranch main
```

## 第二步：克隆项目并启动

### 2.1 克隆代码

```bash
# 进入你的开发目录
cd ~/Projects

# 克隆项目（替换为实际仓库地址）
git clone git@github.com:your-org/yian-shop.git
cd yian-shop
```

### 2.2 一键启动开发环境

```bash
# 给脚本执行权限（Git Bash / WSL）
chmod +x scripts/setup-dev.sh

# 启动完整版（推荐，你的 32GB 内存完全够用）
./scripts/setup-dev.sh full
```

脚本会自动完成：
1. ✅ 检查 Docker 环境
2. ✅ 创建 `.env` 文件
3. ✅ 构建 Docker 镜像
4. ✅ 启动所有容器
5. ✅ 等待 MySQL 就绪
6. ✅ 安装 Composer 依赖
7. ✅ 运行数据库迁移和种子数据

**首次启动约需 5-10 分钟**（主要耗时在下载镜像和构建）。

### 2.3 验证启动成功

```bash
# 查看运行中的容器
docker compose -f docker-compose.dev.yml ps

# 预期输出：
# NAME                STATUS
# yian-dev-app        running
# yian-dev-nginx      running
# yian-dev-mysql      running (healthy)
# yian-dev-redis      running
# yian-dev-meilisearch running
# yian-dev-queue      running
# yian-dev-mailpit    running
```

浏览器访问：
- http://localhost:8080/health → 应返回 `healthy`
- http://localhost:8080/api/v1/health → 应返回 JSON `{"status":"ok"}`

## 第四步：日常开发工作流

### 4.1 常用命令速查

```bash
# ---------- 容器操作 ----------
# 查看所有容器状态
docker compose -f docker-compose.dev.yml ps

# 查看日志（实时）
docker compose -f docker-compose.dev.yml logs -f app
docker compose -f docker-compose.dev.yml logs -f mysql

# 进入 PHP 容器（运行 Artisan 命令）
docker compose -f docker-compose.dev.yml exec app bash

# 重启某个服务
docker compose -f docker-compose.dev.yml restart app

# 停止所有服务（保留数据）
docker compose -f docker-compose.dev.yml stop

# 停止并删除容器（数据保留在 Volume 中）
docker compose -f docker-compose.dev.yml down

# 彻底重置（删除所有数据和容器，慎用！）
docker compose -f docker-compose.dev.yml down -v

# ---------- Artisan 命令 ----------
# 创建控制器
docker compose -f docker-compose.dev.yml exec app php artisan make:controller Api/OrderController

# 创建模型 + Migration + Factory
docker compose -f docker-compose.dev.yml exec app php artisan make:model Order -mf

# 运行迁移
docker compose -f docker-compose.dev.yml exec app php artisan migrate

# 刷新数据库（回滚所有迁移并重新运行）
docker compose -f docker-compose.dev.yml exec app php artisan migrate:fresh --seed

# 运行测试
docker compose -f docker-compose.dev.yml exec app php artisan test

# Tinker 交互式 Shell
docker compose -f docker-compose.dev.yml exec app php artisan tinker

# ---------- Composer ----------
docker compose -f docker-compose.dev.yml exec app composer install
docker compose -f docker-compose.dev.yml exec app composer update

# ---------- 数据库 ----------
# 进入 MySQL
docker compose -f docker-compose.dev.yml exec mysql mysql -uyian -pyian yian_shop

# 进入 Redis
docker compose -f docker-compose.dev.yml exec redis redis-cli
```

### 4.2 Git 工作流

```bash
# 1. 创建功能分支
git checkout -b feature/order-payment

# 2. 开发代码...
# 修改文件后

# 3. 本地测试通过
docker compose -f docker-compose.dev.yml exec app php artisan test

# 4. 提交代码
git add .
git commit -m "feat: 实现订单支付回调处理"

# 5. 推送到远程
git push origin feature/order-payment

# 6. 在 GitHub/GitLab 发起 Pull Request

# 7. Code Review 通过后合并到 develop

# 8. 合并后自动触发 Staging 部署（如已配置 CI）
# 或手动部署: ./scripts/deploy-staging.sh
```

## 第五步：Staging 部署

### 5.1 首次配置服务器

SSH 登录到你的阿里云服务器：

```bash
ssh root@47.121.199.11
```

在服务器上安装 Docker：

```bash
# 更新系统
apt update && apt upgrade -y

# 安装 Docker
curl -fsSL https://get.docker.com | sh

# 安装 Docker Compose
apt install -y docker-compose-plugin

# 验证
docker --version
docker compose version

# 创建部署目录
mkdir -p /opt/yian-staging
```

### 5.2 配置 SSH 免密登录（本地机器执行）

```bash
# 生成本地 SSH 密钥（如果没有）
ssh-keygen -t ed25519 -C "your-email@example.com"

# 复制公钥到服务器
ssh-copy-id root@47.121.199.11
```

### 5.3 一键部署

```bash
# 在项目根目录执行
chmod +x scripts/deploy-staging.sh
./scripts/deploy-staging.sh
```

首次部署后，**必须登录服务器编辑 `.env` 文件**，设置正确的 `APP_KEY`：

```bash
ssh root@47.121.199.11
cd /opt/yian-staging

# 生成 APP_KEY
php -r "echo 'APP_KEY=' . base64_encode(random_bytes(32)) . PHP_EOL;"

# 编辑 .env，替换 APP_KEY
nano .env

# 重启容器
docker compose -f docker-compose.staging.yml restart
```

### 5.4 验证 Staging

浏览器访问 http://47.121.199.11 应能看到应用。

## 第六步：Xdebug 调试配置（VS Code）

### 6.1 安装 VS Code 扩展

安装 **PHP Debug** (xdebug.php-debug)

### 6.2 配置 launch.json

在项目根目录创建 `.vscode/launch.json`：

```json
{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "Listen for Xdebug (Docker)",
            "type": "php",
            "request": "launch",
            "port": 9003,
            "pathMappings": {
                "/var/www/html": "${workspaceFolder}"
            },
            "xdebugSettings": {
                "max_children": 128,
                "max_data": 512,
                "max_depth": 3
            }
        }
    ]
}
```

### 6.3 开始调试

1. 在 VS Code 左侧点击「运行和调试」图标
2. 选择 "Listen for Xdebug (Docker)" 并按 F5
3. 在代码中设置断点（点击行号左侧）
4. 浏览器访问对应 API，VS Code 会自动停在断点处

## 第七步：常见问题排查

### Q1: Docker 启动报错 "port is already allocated"

```bash
# 查看哪个进程占用了端口
netstat -ano | findstr :8080

# 或者更换本地端口
# 编辑 docker-compose.dev.yml，将 nginx 的 ports 改为 "8081:80"
```

### Q2: MySQL 启动失败 "Permission denied"

```bash
# Windows WSL2 下文件权限问题
docker compose -f docker-compose.dev.yml down -v
docker compose -f docker-compose.dev.yml up -d
```

### Q3: Composer 安装很慢

```bash
# 更换阿里云镜像
docker compose -f docker-compose.dev.yml exec app composer config -g repo.packagist composer https://mirrors.aliyun.com/composer/
```

### Q4: 前端 API 请求跨域报错

`.env` 中已配置 `SANCTUM_STATEFUL_DOMAINS=localhost:5173,localhost:8080`，一般不会有问题。如果还有问题，检查前端 `vite.config.ts` 中的 proxy 配置。

### Q5: 内存不足（如果电脑 < 16GB）

```bash
# 使用精简版
./scripts/setup-dev.sh lite
```

### Q6: Staging 服务器 MySQL OOM

```bash
# 服务器上检查内存使用
ssh root@47.121.199.11 "free -h"

# 如果内存吃紧，重启容器
docker compose -f docker-compose.staging.yml restart

# 或者进一步限制 MySQL 内存（已配置 768M 上限，通常够用）
```
