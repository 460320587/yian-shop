## 1. 通用配置规范

### 1.1 环境标识

| 环境 | APP_ENV | 域名 | 说明 |
|------|---------|------|------|
| 本地开发 | `local` | `http://localhost:3000` / `http://localhost` | 开发者本地 |
| 开发服务器 | `development` | `https://dev.yian.com` | 团队共享开发环境 |
| 测试环境 | `testing` | `https://test.yian.com` | QA测试 |
| 预发布 | `staging` | `https://staging.yian.com` | 上线前验收 |
| 生产环境 | `production` | `https://www.yian.com` | 正式环境 |

### 1.2 配置优先级（Laravel）

```
.env > .env.{APP_ENV} > 数据库配置表(system_configs) > config/*.php默认值
```

### 1.3 命名规范

- 全大写，下划线分隔
- 布尔值: `true` / `false`（字符串）
- 数组: 逗号分隔，如 `CACHE_STORE=redis,database`
- 路径: 绝对路径或相对于项目根目录

## 5. 第三方服务配置说明

### 5.1 微信支付配置获取流程

1. **申请商户号**: 微信支付商户平台 → 提交企业资质 → 审核(2-4周)
2. **获取API证书**: 商户平台 → API安全 → 申请API证书 → 下载`apiclient_cert.pem` + `apiclient_key.pem`
3. **设置APIv3密钥**: 商户平台 → API安全 → 设置APIv3密钥 → 32字节随机字符串
4. **配置回调地址**: 商户平台 → 产品中心 → 开发配置 → 添加回调URL

### 5.2 支付宝配置获取流程

1. **创建应用**: 支付宝开放平台 → 创建网页/移动应用
2. **配置应用**: 应用详情 → 接口加签方式 → 生成密钥对
3. **下载公钥/私钥**: 保存应用私钥 + 支付宝公钥
4. **配置回调地址**: 应用详情 → 应用网关/授权回调地址

### 5.3 短信服务配置

**阿里云短信**:
1. 阿里云控制台 → 短信服务 → 开通服务
2. 申请短信签名（企业名称，1-2周审核）
3. 申请短信模板（15个模板CODE）
4. 获取AccessKey ID/Secret

**短信模板清单**（已在主文档Phase 8定义）:
- 验证码类5个: 注册/登录/找回密码/修改手机/支付密码重置
- 通知类8个: 支付成功/发货/完成/退款/VIP升级/降级/积分过期/优惠券过期
- 营销类2个: 促销活动/生日祝福

### 5.4 物流接口配置

**快递鸟**:
1. 快递鸟官网注册企业账号
2. 获取API Key和Secret
3. 申请电子面单接口权限
4. 配置月结账号（各承运商）

### 5.2 回滚脚本 (rollback.sh)

```bash
#!/bin/bash
set -e

TIMESTAMP=$1
BACKUP_DIR="/backup/yian/${TIMESTAMP}"
DEPLOY_DIR="/var/www/yian"

if [ -z "$TIMESTAMP" ]; then
    echo "用法: ./rollback.sh <timestamp>"
    echo "可用备份:"
    ls -la /backup/yian/
    exit 1
fi

echo "⏮️ 开始回滚到: ${TIMESTAMP}"

# 1. 恢复数据库
echo "🗄️ 恢复数据库..."
mysql -u root -p${DB_PASSWORD} yian_shop < ${BACKUP_DIR}/database.sql

# 2. 恢复代码
echo "📦 恢复代码..."
cd ${DEPLOY_DIR}
tar xzf ${BACKUP_DIR}/code.tar.gz

# 3. 清除缓存
php artisan config:cache
php artisan route:cache
php artisan view:cache
php artisan octane:reload
php artisan horizon:terminate

echo "✅ 回滚完成"
```

### 5.3 蓝绿部署脚本 (blue-green-deploy.sh)

```bash
#!/bin/bash
# 蓝绿部署：保持旧版本运行，验证新版本后再切换流量

ENV=$1
CURRENT_COLOR=$(cat /var/www/yian/.color 2>/dev/null || echo "green")
NEW_COLOR=$([ "$CURRENT_COLOR" = "green" ] && echo "blue" || echo "green")

echo "当前环境: ${CURRENT_COLOR}"
echo "新版本将部署到: ${NEW_COLOR}"

# 部署到非活跃环境
DEPLOY_DIR="/var/www/yian-${NEW_COLOR}"
# ... 执行部署

# 健康检查
if curl -f https://yian-${NEW_COLOR}.internal/health; then
    # 切换Nginx upstream
    sed -i "s/server app-${CURRENT_COLOR}/server app-${NEW_COLOR}/" /etc/nginx/conf.d/upstream.conf
    nginx -s reload
    
    echo ${NEW_COLOR} > /var/www/yian/.color
    echo "✅ 流量已切换至 ${NEW_COLOR}"
else
    echo "❌ 新版本健康检查失败，保持 ${CURRENT_COLOR} 运行"
    exit 1
fi
```

## 3. 阶段详解

### 3.1 Build阶段 (3-5分钟)

| 任务 | 说明 | 输出 |
|------|------|------|
| `build_backend` | `composer install` + 优化自动加载 | `vendor/`目录Artifact |
| `build_frontend` | `npm ci` + `npm run build` | `resources/js/dist/` Artifact |

**优化**:
- Composer使用`--no-dev`减少体积
- npm使用`--cache`加速重复构建
- Artifact在后续阶段通过`needs`传递

### 3.2 Test阶段 (5-10分钟)

| 任务 | 类型 | 目标 | 并行 |
|------|------|------|:----:|
| `test_unit` | Pest单元测试 | 覆盖率≥80% | ✅ |
| `test_integration` | API集成测试 | 全部通过 | ✅ |
| `test_e2e` | Cypress E2E | 核心旅程通过 | ✅ |

**测试数据库**: MySQL 8.0服务容器，每次构建独立实例  
**Redis**: 服务容器，用于队列/缓存/会话测试  
**Meilisearch**: 服务容器，用于搜索测试

### 3.3 Lint阶段 (2-3分钟)

| 任务 | 工具 | 级别 | 阻断 |
|------|------|------|:----:|
| `lint_php` | Laravel Pint | 代码风格 | ✅ |
| `lint_phpstan` | PHPStan Level 8 | 静态分析 | ✅ |
| `lint_frontend` | ESLint + TypeScript | 代码规范 | ✅ |

**Pint配置** (`pint.json`):
```json
{
  "preset": "laravel",
  "rules": {
    "ordered_imports": {"sort_algorithm": "alpha"},
    "no_unused_imports": true,
    "array_indentation": true
  }
}
```

**PHPStan配置** (`phpstan.neon`):
```neon
includes:
    - vendor/larastan/larastan/extension.neon

parameters:
    level: 8
    paths:
        - app
        - routes
        - tests
    excludePaths:
        - app/Console/Kernel.php
    checkModelProperties: true
    checkMissingIterableValueType: false
```

### 3.4 Security阶段 (3-5分钟)

| 任务 | 工具 | 检测内容 | 阻断 |
|------|------|---------|:----:|
| `security_composer_audit` | Composer Audit | PHP依赖CVE | ✅ |
| `security_npm_audit` | npm audit | JS依赖CVE | ✅ |
| `security_trivy` | Trivy | 文件系统漏洞 | ✅ |
| `security_secrets_scan` | Gitleaks | 密钥泄露 | ✅ |
| `security_sonarqube` | SonarQube | 代码质量漏洞 | ⚠️ (不阻断) |

**安全阈值**:
- High/Critical漏洞 = 阻断构建
- Medium/Low漏洞 = 告警但不阻断（SonarQube）

### 3.5 Deploy阶段 (2-5分钟)

| 任务 | 触发条件 | 环境 | 方式 |
|------|---------|------|------|
| `deploy_staging` | `develop`分支合并后自动 | Staging | SSH + 脚本 |
| `deploy_production` | `main`分支合并后手动触发 | Production | SSH + 脚本 |

**生产部署保护**:
- 需要手动点击"Play"按钮
- 需要特定GitLab用户权限
- 部署窗口限制：工作日 02:00-06:00
