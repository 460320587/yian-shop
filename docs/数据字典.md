## 1. 命名规范

### 1.1 表命名
- 全小写，下划线分隔，复数形式
- 关联表: `{表A}_{表B}` 按字母序，如 `role_permission`
- 日志/记录表: `{主体}_logs` / `{主体}_records`

### 1.2 字段命名

| 类型 | 命名规则 | 示例 |
|------|---------|------|
| 主键 | `id` (自增BIGINT UNSIGNED) | `id` |
| 外键 | `{关联表单数}_id` | `user_id`, `order_id` |
| 状态 | `status` (tinyint/string) | `status` |
| 类型 | `type` / `{实体}_type` | `order_type`, `invoice_type` |
| 金额 | `amount` / `{业务}_amount` | `total_amount`, `refund_amount` |
| 时间戳 | `created_at` / `updated_at` / `deleted_at` (软删除) | Laravel默认 |
| 操作人 | `created_by` / `updated_by` | 后台记录操作用户 |
| 排序 | `sort` / `display_order` | `sort` (默认0，越大越靠前) |
| 布尔 | `is_{形容词}` / `has_{名词}` | `is_default`, `is_recommended` |

### 1.3 通用字段（所有表必须包含）

| 字段 | 类型 | 默认值 | 说明 |
|------|------|:------:|------|
| `id` | BIGINT UNSIGNED | auto_increment | 主键 |
| `created_at` | TIMESTAMP | CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | TIMESTAMP | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

## 附录: 索引规范速查

### 必须加索引的场景
1. 外键字段
2. WHERE条件常用字段
3. ORDER BY字段
4. 状态/status字段(如果区分度>0.1)
5. 时间范围查询字段
6. 全文搜索字段

### 避免索引的场景
1. 区分度极低的字段(如gender=1/2)
2. 频繁更新的字段(考虑索引维护成本)
3. 大字段(TEXT/BLOB不加普通索引)
4. 小表(<1000行)

### 索引命名规范
- 主键: `PRIMARY`
- 唯一索引: `uk_{表名}_{字段名}`
- 普通索引: `idx_{表名}_{字段名}`
- 联合索引: `idx_{表名}_{字段1}_{字段2}`
- 全文索引: `ft_{表名}_{字段名}`
